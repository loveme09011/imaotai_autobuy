# 自动申购核心逻辑

## 背景与目标
在每天 09:00:00 精确触发申购请求，支持多账号并发，最大化申购成功率。

## 设计方案
- 数据模型：无持久化，申购结果记录到日志
- 业务流程：
  1. 启动时加载所有账号配置
  2. 查询目标商品 ID 和最优门店（出货量最大）
  3. 时钟同步（NTP 校准本地时间偏差）
  4. 等待至 08:59:59.5，多账号并发发起申购
  5. 记录每个账号的申购结果
- 依赖关系：依赖 `api/client.py`、`core/account.py`、`utils/time_sync.py`

## 边界与约束
- 性能要求：多账号并发使用 `asyncio` 或 `ThreadPoolExecutor`
- 安全考量：申购失败不重试（避免触发风控），记录原因即可
- 已知限制：每账号每天只能申购一次，中签率由系统决定

# 申购结果通知

## 背景与目标
申购完成后，将每个账号的申购结果（成功/失败/原因）推送给用户，无需手动查看日志。

## 设计方案
- 通知渠道：优先支持 Server酱（微信推送），可扩展其他渠道
- 业务流程：申购任务完成 → 汇总所有账号结果 → 格式化消息 → 推送
- 依赖关系：依赖 `core/reservation.py`

## 边界与约束
- 安全考量：推送 key 存配置文件，不硬编码
- 已知限制：依赖第三方推送服务可用性

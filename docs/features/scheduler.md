# 定时调度

## 背景与目标
每天定时在 09:00:00 自动触发申购，并在非申购时间段维护账号耐力值（自动旅行）。

## 设计方案
- 业务流程：
  - 09:00:00 → 触发申购任务
  - 每日随机时间（非9点）→ 触发旅行任务（维护耐力值）
  - 程序常驻运行，使用 `schedule` 库管理任务
- 依赖关系：依赖 `core/reservation.py`、`api/client.py`

## 边界与约束
- 性能要求：申购任务优先级最高，旅行任务不影响申购
- 安全考量：旅行时间随机化，避免固定规律触发风控
- 已知限制：需要程序 24/7 运行（建议部署在服务器）
